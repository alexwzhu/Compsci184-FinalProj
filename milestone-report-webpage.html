Summary

We're building a real-time 3D Gaussian Splatting renderer in WebGPU. The input is a pre-trained .ply file full of Gaussians, and the output is pixels on a canvas. The parser, 3D→2D projection, sorting, SH evaluation, and alpha compositing in between are all code we wrote. Right now we have a working CPU reference renderer. It loads real trained data, projects and composites the Gaussians correctly, and you can orbit and fly around with the camera.

Scope: What We Build vs. What We Consume

We're building on top of existing work, so it helps to be clear about where the line is. The .ply files we load are pre-trained. Someone else's trainer (INRIA, gsplat, or nerfstudio) ran gradient descent against real photographs to produce them, and we don't touch any of that training code. It's basically the HW3 setup. HW3 loaded .dae files that Blender or Maya had already produced, and we load .ply files that a trainer already produced. Everything between reading the file and putting pixels on screen is ours.

Consumed (not ours)

  • Pre-trained .ply Gaussian parameters from INRIA trainer
  • COLMAP camera-pose recovery (upstream of training)
  • Mip-NeRF 360 captured-photograph dataset

Built from scratch (ours)

  • Binary .ply parser with sigmoid/exp activations
  • Frustum culling & 3D→2D EWA covariance projection
  • Tile binning + bitonic depth sort (WebGPU / WGSL)
  • SH evaluation (up to degree 2) & alpha compositing
  • CPU reference renderer, interactive camera, PSNR/SSIM tooling

What's Working So Far

  • PLY parser: reads the full INRIA binary format, including position, scale, rotation quaternion, opacity, and all 48 SH coefficients per Gaussian.
  • 3D→2D projection on CPU: EWA covariance projection Σ' = J W Σ WT JT, with the Jacobian evaluated per-Gaussian in camera space.
  • Alpha compositing: front-to-back blending per pixel, transmittance accumulation, and early termination once T drops below 1/255.
  • Camera: orbit (click-drag), zoom (scroll), pan (right-click-drag), WASD fly-through, and auto-fit to the scene on load.
  • Tested on real data: a 14,526-Gaussian Luigi model from an actual trainer renders end-to-end, not just our synthetic scenes.

Progress Reflection

We're roughly on schedule. The CPU side (projection, compositing, camera) is working and the output matches what we expect visually. Our biggest headache was the covariance projection. Deriving the Jacobian from the Zwicker EWA paper was the easy part, but we got the matrix-multiplication order wrong a couple of times (row-major vs. column-major confusion) and spent a while debugging blobs that came out squished in weird directions. Now that the CPU renderer works, porting to the GPU feels a lot less scary, because we can diff our compute-shader output against it pixel by pixel. Weeks 3 and 4 are the WebGPU compute shaders (tile binning, bitonic sort, SH degree 2, and alpha compositing), plus loading the Mip-NeRF 360 scenes with over a million Gaussians each, and running PSNR/SSIM against the INRIA reference renderer for the final report.
