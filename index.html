Splat the Web: A Real-Time WebGPU Renderer for 3D Gaussian Splatting

CS 184: Computer Graphics and Imaging | Spring 2026 Final Project Proposal
Team: Ben Shamloufard, Kaan Kont, Alex Zhu
Repo: github.com/alexwzhu/Compsci184-FinalProj

Summary

Splat the Web is a real-time 3D Gaussian Splatting renderer written in WebGPU. We take pre-trained Gaussian models from existing trainers and build the entire rendering engine from scratch, so anyone with a modern browser can fly through photorealistic scans of real scenes.

Problem Description

Given a set of photos of a real scene from many angles, we want to reconstruct it as a 3D representation and render new views of it in a browser at interactive frame rates. 3D Gaussian Splatting (Kerbl et al., SIGGRAPH 2023) does this by representing the scene as millions of semi-transparent 3D Gaussians. Each one stores a position, a covariance matrix (a scale and a rotation quaternion), an opacity, and spherical harmonic coefficients for view-dependent color. Because Gaussians can be rasterized rather than ray-marched, the technique runs above 100 FPS at 1080p. Comparable NeRFs take seconds per frame.

Our project is the renderer, not the trainer. We use existing open-source pipelines (the original INRIA code, gsplat, or nerfstudio) to pre-train Gaussian models offline. The same way Project 3 loaded .dae files that other tools produced, we load .ply files of trained Gaussians and turn them into pixels. Every line of code that does that conversion is ours.

The hard parts are getting the math right and getting the GPU pipeline fast. Each 3D covariance has to be projected to a 2D screen-space ellipse using a Jacobian that goes unstable near the camera. Millions of Gaussians have to be depth-sorted per tile every frame, which means writing a parallel sort, prefix sums, and tile binning in WebGPU compute shaders by hand. To stay honest about correctness, we also build a slow but readable CPU reference renderer in TypeScript. Any disagreement between the GPU output and the reference flags a bug.

Goals and Deliverables

What we plan to deliver

  • WebGPU renderer that loads pre-trained .ply Gaussian models and runs at 30+ FPS at 1080p on a modern laptop.
  • Full pipeline written from scratch: .ply parser, frustum culling, 3D-to-2D Gaussian projection, tile binning, GPU bitonic sort, spherical harmonics up to degree 2, and front-to-back alpha compositing.
  • Three scenes from the Mip-NeRF 360 dataset (garden, bicycle, room) running in our viewer.
  • CPU reference renderer for correctness checking.
  • Interactive camera with orbit and WASD fly-through.
  • Live web demo on GitHub Pages.
  • Results table with PSNR / SSIM versus the INRIA reference, plus FPS at 720p, 1080p, and 4K.

What we hope to deliver, in priority order

  1. GPU radix sort replacing bitonic.
  2. Spherical harmonics degree 3 for fine specular effects.
  3. Mip-splatting style anti-aliasing (Yu et al., CVPR 2024).
  4. Gaussian compression with a quality vs file size analysis.
  5. A custom Berkeley scene captured on phones, processed through COLMAP and a Gaussian trainer.
